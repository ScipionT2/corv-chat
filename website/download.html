Nova (Free)

The full local AI assistant. Voice control, screen analysis, and a beautiful UI — all running on your Mac, all offline.

🍎 macOS ⚡ Apple Silicon recommended 💾 8GB+ RAM 📦 ~4GB disk space
  1. Install prerequisites
    brew install python3 ollama
  2. Clone Nova
    git clone https://github.com/escipionpedroza147-commits/Nova.git cd Nova
  3. Set up environment
    python3 -m venv .venv source .venv/bin/activate pip install -r requirements.txt
  4. Download the AI model
    ollama pull qwen2.5:3b
  5. Launch Nova
    python3 ep_agent.py
💡 Say "Nova" to start talking! The sidebar appears on the right side of your screen.

Nova Control Hub

Web dashboard to monitor and control Nova. View conversations, adjust settings, and manage your assistant from any browser.

✅ Included with Nova — starts automatically when you launch the app!
  1. Make sure Nova is running
    Launch Nova with python3 ep_agent.py — the Control Hub starts with it.
  2. Open your browser
    http://localhost:8766
  3. That's it!
    The dashboard loads automatically. Monitor conversations, tweak settings, and control Nova from any device on your network.
⚙️ Use --no-web to disable the dashboard, or --web-port 9000 to use a custom port.

Troubleshooting

Having issues?

Most problems have quick fixes. Check below or open an issue on GitHub.

Run ollama serve in a separate terminal window. This starts the Ollama server manually. If it says the port is already in use, Ollama is likely already running in the background — you're good to go.
The first model download (~2GB for qwen2.5:3b) can take a few minutes depending on your connection. This is a one-time download. Subsequent launches are instant since the model is cached locally.
Make sure your microphone permissions are granted. Go to System Preferences → Privacy & Security → Microphone and ensure your terminal app (or Python) has access. You may need to restart Nova after granting permissions.
Nova requires Python 3.10 or later. Check your version with python3 --version. If you're on an older version, update via Homebrew: brew upgrade python3. Make sure you're using the correct Python when creating the virtual environment.
Check the GitHub Issues page for known bugs and solutions. If your problem isn't listed, open a new issue with your system info and error message — we'll help you out.
